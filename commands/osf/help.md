# OpenSpec-Flow Help

OpenSpec-Flow bridges OpenSpec (specification-driven change management) with Claude Flow (multi-agent orchestration) via Claude Code.

## Installation

```bash
npm install -g openspec-flow
openspec-flow setup
```

### Required: Claude-Flow MCP

The `/osf:implement`, `/osf:verify`, and `/osf:review` commands require Claude-Flow:

```bash
claude mcp add claude-flow -- npx @anthropic/claude-flow@alpha mcp start
```

## Commands

| Command | Description | Requires Claude-Flow |
|---------|-------------|---------------------|
| `/osf:ideate <req>` | Create new change from requirements | No |
| `/osf:list` | List all OpenSpec changes | No |
| `/osf:work <id>` | Generate work brief | No |
| `/osf:analyze <id>` | Analyze change size/complexity | No |
| `/osf:split <id>` | Split large change into phases | No |
| `/osf:implement <id>` | Implement via agent swarm | **Yes** |
| `/osf:verify <id>` | Verify via test agents | **Yes** |
| `/osf:review <id>` | Review via review agents | **Yes** |
| `/osf:deferred <id>` | Analyze incomplete tasks | No |
| `/osf:log <id>` | Create flow log | No |
| `/osf:archive <id>` | Archive completed/closed change | No |

## Workflow

```
1. /osf:ideate "feature"  - Create new change from requirements
   OR manually create openspec/changes/<id>/

2. /osf:list              - See available changes
3. /osf:analyze CHANGE-001 - Check size/complexity
4. /osf:split CHANGE-001   - Split if too large (optional)
5. /osf:work CHANGE-001    - Generate work brief
6. /osf:implement CHANGE-001 - Implement via swarm orchestration
7. /osf:verify CHANGE-001  - Verify via test agents
8. /osf:review CHANGE-001  - Review via review agents
9. /osf:deferred CHANGE-001 - Check incomplete items
10. /osf:log CHANGE-001    - Document implementation
11. /osf:archive CHANGE-001 - Archive when done
```

## Architecture

```
/osf:implement CHANGE-001
    │
    ▼
OpenSpec-Flow MCP (get_change_context)
    │
    ▼
Claude-Flow MCP (swarm_init, agents_spawn_parallel, task_orchestrate)
    │
    ▼
Spawned Agents (do actual implementation work)
```

## Claude-Flow Integration

The `/osf:implement`, `/osf:verify`, `/osf:review` commands use Claude-Flow's multi-agent orchestration:

1. **swarm_init** - Creates agent swarm (hierarchical/mesh/ring/star topology)
2. **agent_spawn** - Spawns specialized agents (coder, tester, reviewer, etc.)
3. **task_orchestrate** - Passes work brief to agents for execution
4. **task_results** - Collects results from all agents
5. **swarm_destroy** - Gracefully shutdown swarm when done

## OpenSpec Structure

```
openspec/
└── changes/
    ├── <CHANGE-ID>/
    │   ├── proposal.md    # Change proposal
    │   ├── tasks.md       # Implementation tasks
    │   ├── design.md      # Optional design docs
    │   ├── work-brief.md  # Generated by /osf:work
    │   ├── flow-log.md    # Generated by /osf:log
    │   ├── phases.yaml    # Generated by /osf:split (if split)
    │   └── specs/         # Spec delta files
    └── archive/           # Archived changes
        └── YYYY-MM-DD-<CHANGE-ID>/
            ├── ...
            └── archive-metadata.yaml
```

## Troubleshooting

**"claude-flow tools not found"**
Install Claude-Flow MCP:
```bash
claude mcp add claude-flow -- npx @anthropic/claude-flow@alpha mcp start
```

**"OpenSpec directory not found"**
Create the structure:
```bash
mkdir -p openspec/changes
```

**"OpenSpec CLI required"**
Install OpenSpec CLI:
```bash
npm install -g @anthropic/openspec
```
